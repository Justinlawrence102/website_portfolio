LogRide

Theme Park Tracker

I worked on with a small team of developers and researchers to create an application for iPhone. Since the app launched in November 2018, we have had very positive reviews and thousands of users. I am continually working on the app to add new features.

To create LogRide, I taught myself the Swift programing language, as well as various technologies that are used within the app including php, SQL, and Firebase. I also became expererienced in UI design to create a good user experience.

I work as a full stack developer for most of the features I implement within the app. I begin by creating multiple mockups of what the design could look like, and tweak the concepts based on feedback from the rest of the team. After that, depending on what the feature requires, I write SQL code to interact with our backend data base as well as implement all of the UI and UX of the design.

User Feedback Loop

User submitted attractions and photos has been a major componment since we started LogRide, but until recently, there was no way for users to receive feedback after they submitted something. To create this feature, I used advanced SQL commands to join data from multiple talbes to display to the user their previous submissions, as well as feedback the LogRide team provides.

LogRide Pro

To help offset the costs of operating an application, I created a subscription based 'pro' version of LogRide. To support payment within the app, I had to carefully follow Apple's guidlines to make sure it would be approved for the App Store. To implement new pro features I got exerience integrating 3rd party APIs into LogRide by adding features like the Backdating Calender.

Stats

After LogRide has been on the App Store for a year, we noticed that the feature was lacking the most was how stats were being displayed. To improve this feature, I re-wrote it with a focus to make stats more detailed and cusomizable.
