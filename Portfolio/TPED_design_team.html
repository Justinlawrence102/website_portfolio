Gator Theme Park Engineering & Design

Model Roller Coaster Control System

For the past 2 semesters, the Gator Theme Park Engineering & Design club has been working on a model roller coaster for out design project. In the Spring of 2019, we designed and 3D printed a ride vehicle. Then, over the past year, we have split the design team into mechanical, civil, and controls subgroups. The final project will incorparte a 3D printed track layout, a ride vechicle, and a functional ride control system

I have been leading the software/hardware subteam for the past year. Our task was to design a control system that will allow two trains to be running on the track and ensure they will not hit eachother. We first deisgned the logic of the control system by using a Petri Net diagram. I leanred about this type of logic diagram in my Embedded Systems class, and it is a visual way to ensure that two ride vehicles will not be on the same segment of track at the same time.

Once we finizlied the logic diagram, we coded the control system using an arduino. We are using IR sensors to keep track of which block of track the ride vehicles are located in. Once the track layout is complete, we will solder all of the wires and build a control box to house the main control buttons. We followed ASTM F24 standards when designing the control panel to make ours similar to what they look like in the indusrty.

The model will be able to operate in three different modes. First, there is automatic or disply mode. This will allow the model to run by itself, and dispatch a new ride vehicle ever couple of minutes. The next mode is manual mode, which the user will be able to click a button to dispatch the vehicles. Finally if the user has the operation key, they can operate in maaintnance mode. This will override the control system and allow each of the motors be controlled individually.
