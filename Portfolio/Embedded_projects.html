In the Spring semester of 2020, I took a class on embedded software. The class focused on a wide range of topics in this catagory including artectecture, optimizations, security, and rtos. Using what I learned in this class, I completed two projects using C++ that simulate tasks often completed on embeded computers.

The task of the first project was to write a simulation of a simple ARM processor using C++. There were 16 registers and 16 memory locations, and each instruction held 16 bits. Based on the OP code, the simulator could preform the following tasks: Read, Decode, Add, Multiply, Calculate address, Store, and Write. It was written dynamically, so you could have any number of instructions. The simulation would print out what values were in each of the registers and data path.

View the GitHub Page: https://github.com/Justinlawrence102/embeddedSystemP1

For second project, I created a C++ program that preforms a compression and decompression algorithum a set of instructions in machine code. To compress the input code as much as possible, I used a combination of compression techniques. I created a dictionary to hold the most common instructions, and used bitmasking and 2 bit mismatching to compresses the rest of the instructions. Then, to preform the decompression, I worked backwards to get the original instruction set back.
